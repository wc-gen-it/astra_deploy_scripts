initial_setup_common.yml и software_setup_common.yml:
эти файлы являются так называемыми плейбуками или исполняемыми сценариями для awx.
В них содержится директивы для запуска той или иной роли/задачи.

Роль это совокупность исполняемых команд на выбранных машинах.
В данном случае роль - "общая"(common) и в контексте awx проекта, она обладает своими задачами(tasks), переменными (defaults, vars) и хендлерами (handlers).

roles/*role*/vars & roles/*role*/defaults
разница между vars и defaults лишь в том, что переменные указанные в vars считываются после defaults и при наличии одной переменной в каждой из этих директорий
ей будет присвоено значение указанное в vars на этапе выполнения сценария.
Кроме того, есть возможность переопределять значения переменных непосредственно из веб-интерфейса awx.
Переменные заданные в awx внутри шаблона (template) будут иметь "самый высокий приоритет".

roles/*role*/tasks
содержит задачи специфичные для роли.
При вызове роли из плейбука по умолчанию запускается файл tasks/main.yml.

roles/*role*/handlers
это так называемые хендлеры. Они запускаются по ключевому слову notify в задаче (task).
Наличие хендлеров позволяет строить гибкие сценарии и избежать переизбытка задач внутри роли.
Директива в виде notify: some command по умолчанию запустит поиск хендлера в handlers/main.yml.
Такое поведение может быть переопределено.

group_vars и host_vars
содержат переменные, специфичные для какой-либо группы или хоста.
название файла содержащего переменные должно соотноситься с названием группы/хоста.

shared
архитектурно не имеет никакого отношения ни к ansible ни к awx
создана в качестве хранилища для прочих файлов.
